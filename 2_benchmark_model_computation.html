2) Benchmark Model Computation¶

As mentioned in the proposal, we are going to use a very simple indicator - Moving Average Convergence Divergence (MACD), to be our benchmark model for comparison. The formula of MACD is:

$\text{MACD} = \text{EMA}_{12}(\text{price}) - \text{EMA}_{26}(\text{price})$
$\text{Signal} = \text{EMA}_{9}(\text{MACD})$
$\text{Divergence}_t = \text{MACD}_t - \text{Signal}_t$
$\text{Prediction} = \begin{cases} \text{Up} & \text{if } \text{Divergence}_t > 0 \text{ and } \text{Divergence}_{t - 1} < 0 \\ \text{Down} & \text{if } \text{Divergence}_t < 0 \text{ and } \text{Divergence}_{t - 1} > 0 \end{cases}$

Therefore, in this notebook, we will compute the signals and prediction generated by MACD, calculate its prediction accuracy, and store it locally in our project files for future uses

# MACD = EMA12(close) - EMA26(close)
df.loc[:, 'MACD'] = df['close'].ewm(span=12).mean() - df['close'].ewm(span=26).mean()

# Signal = EMA9(MACD)
df.loc[:, 'Signal'] = df['MACD'].ewm(span=9).mean()

# Divergence = MACD - Signal
df.loc[:, 'Divergence'] = df['MACD'] - df['Signal']

Benchmark Evaluation¶

We can then define rules to measure accuracy of the predictions by MACD. As we mentioned before:

$\text{Prediction} = \begin{cases} \text{Up} & \text{if } \text{Divergence}_t > 0 \text{ and } \text{Divergence}_{t - 1} < 0 \\ \text{Down} & \text{if } \text{Divergence}_t < 0 \text{ and } \text{Divergence}_{t - 1} > 0 \end{cases}$

We can split the predictions into 3 different windows: 1-minute (short-term), 10-mintue (medium-term) and 60-minute (long-term) price change direction. Under 3 different windows, we can further define the predictions to be:

$\text{Let } x \text{ be the specific length of window}$
$\text{Forward } x \text{-minute Prediction} = \begin{cases} \text{Up} & \text{if } \text{Divergence}_t > 0 \text{ and } \text{Divergence}_{t - 1} < 0 \\ \text{Down} & \text{if } \text{Divergence}_t < 0 \text{ and } \text{Divergence}_{t - 1} > 0 \end{cases}$

Then we will calculate the accuracy of each case to evaluate how well this benchmark model performs. Since it is not a binary classification problem, normal meaures (precision & recall) may not work here. So we will use 2 custom metrics to evaluate its performance:

$\text{Prediction Frequency} = \frac{\text{Total number of predictions made}}{\text{Total number of price changes}}$
$\text{Prediction Accuracy} = \frac{\text{Correct predictions}}{\text{Total number of predictions made}}$

Prediction Frequency can be used to measure whether a model is giving too infrequent signal that may be not insightful enough in our case.

# First compute predictions from MACD signals
df.loc[:, 'prediction'] = [0.0, 0.0] + np.sign(np.diff(np.sign(df['Divergence']))).tolist()[1:]

# Then compute the target window forward changes for evaluation
for i in (1, 10, 60):
    df.loc[:, f'{i}m_fwd_chg_sign'] = np.sign(df.close.pct_change(i).shift(-i))

# Then we can evaluate the accuracy of the predictions

result_metrics_dict = {}
for i in (1, 10, 60):
    col = f'{i}m_fwd_chg_sign'
    tmp_dict = {}
    sub_df = df.loc[:, ['prediction', col]].dropna()
    tmp_dict['prediction_frequency'] = sub_df.loc[sub_df['prediction'] != 0, :].shape[0] / sub_df.shape[0]
    tmp_dict['prediction_accuracy'] = sub_df.loc[sub_df['prediction'] == sub_df[col], :].shape[0] / sub_df.loc[sub_df['prediction'] != 0, :].shape[0]
    result_metrics_dict[f'{i}m_result'] = tmp_dict

print(pd.DataFrame(result_metrics_dict))
